    In this thread I will tell you about AutoML üßµüëá

    AutoML stands for Automated Machine Learning.

    It encompasses a bunch of technologies and paradigms to gradually automate the process of creating machine learning solutions.

    üí° AutoML is about raising the abstraction level in ML and reducing the grunt work.

    ‚ùì What can AutoML do today?

    Getting a machine learning solution to work takes a few steps:

    • 1Ô∏è‚É£ collecting data
    • 2Ô∏è‚É£ sanitizing that data
    • 3Ô∏è‚É£ finding the best model
    • 4Ô∏è‚É£ training that model
    • 5Ô∏è‚É£ and beyond, actually build the product‚ùó

    Most current AutoML frameworks today focus on 3Ô∏è‚É£, i.e., helping you select among the plethora of machine learning models which is the best for your problem.

    This problem is often framed in terms of:

    • üÖ∞Ô∏è model selection
    • üÖ±Ô∏è hyperparameter optimization

    üÖ∞Ô∏è Model selection is about deciding, e.g., if logistic regression, decision trees or SVM is better, or whether to encode with word2vec or TF-IDF.

    The "manual" way of doing this is to actually try each algorithm a bunch of times in your data and collect some statistics.

    üÖ±Ô∏è Hyperparameter optimization is about selecting the exact value for each tunable thing in your algorithm.

    • How many neurons?
    • How much dropout?
    • Which activation function?
    • Which regularization factor?
    • ...

    If you combine both problems, then you realize there are literally thousands (and potentially infinite) different algorithms you can try on your data.

    And AutoML is much more than model selection and hyperparameter search. It can also include automating:

    • data preprocessing
    • feature engineering
    • feature selection
    • dataset augmentation
    • model distillation
    • and more...

    üîë If you are working on a practical problem today there is no reason not to use AutoML.

    üîë Even if you are working on research, AutoML will make you more productive by taking care of the dumb tasks and letting you focus on the important parts.

    ‚ùóHowever, this is no silver bullet.

    There are a lot of challenges to make AutoML production-ready. Data cleaning is a major bottleneck still, far from automated. And we need to understand how these methods exacerbate data bias.

    Finally, if you are feeling adventurous, you can try @auto_goal, an experimental AutoML framework that goes beyond "standard" AutoML.
